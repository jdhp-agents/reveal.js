Mathematical optimization

Find the minimum (or the maximum) of a function $f$ $$\boldsymbol{x}^* = \arg\min_{\boldsymbol{x} \in \mathcal{X}} f(\boldsymbol{x})$$

Ok so… to begin with… the basic definition The aim of today's lecture is to study algorithms designed to find either the minimum or the maximum of a function So using this notation [...], the subject here is to find x star.

Minimize or maximize ?


In optimization literature, usually by “optimize” one mean “minimize”

In reinforcement learning by “optimize” one mean “maximize”

Same algorithms in both cases

$$ \arg\min f(x) \Leftrightarrow \arg\max -f(x) $$

The choice between minimization and maximization depends on the problem we try to solve. But, there are some conventions, and... [...] [...] [...] This is not an issue actually, we can use the same algorithms in both cases. We can switch from one convention to the other simply taking the opposite of the function to optimize. So no big news so far but as you will see in the lab, you have to remember this difference of conventions between the optimization and the reinforcement learning communities.

Optimization in AI


Optimization is a main component of modern AI (the other one is Learning)

Optimization and Learning are highly connected

E.g. optimization is used to:

  • set a model in supervised learning (e.g. weights of neural networks)
  • optimize a policy in Reinforcement Learning (Direct Policy Search)

Some optimization algorithms use learning techniques (especially surrogate models)

So you may wonder why I’m talking about optimization in a machine learning course... Well, actually Machine Learning and optimization are the two main topics nowadays in modern IA and these two subjects are totally interconnected and interdependent. For instance the learning procedure in supervised learning is an optimization problem, where we try to find the neuron weights that minimize an error function. Training an agent in reinforcement learning is an optimization problem too, where we try to find the policy’s parameters that maximize the expected reward. So optimization can be seen as a required tool in machine learning. And optimization is often an iterative procedure where we try to find a vector that minimize (or maximize) a function, using solutions what have been seen in the past ; so optimization itself can also be seen as a learning problem! So again, Machine Learning and Optimization are totally interconnected and interdependent and that’s why mathematical optimization is presented here today.

There are many families of optimization problems with differents difficulties… for instance, the objective function can be - [...] - linear or non linear, - convex or non convex - The solution space can be - continuous, when we try to optimize a vector of real values - it can be discrete, when we try to optimize a vector of discrete values (for instance a binary vector) - it can be mixte, when we try to optimize a vector having continuous values on some dimensions and discrete values on others - The objective function can be Unimodal (when only one global optimum to finds or multi-modal (when there are many optima to find) - The problem can be multi-objective (when we try to optimize several objectives at the same time) For instance a robot that tries to reach a target will search a trajectory that minimizes its final distance to the target and minimizes the time to reach the target and minimizes the energy spent for that task In this case, there is a large number of “good” solution with different compromises on each objectives (solutions that are on the “pareto front”) - An analytical formulation of the gradient may be provided or not - The objective function may be stochastic, in this case, it returns different values when it is called several times on the same solution vector. In other words the objective function is noised - The solution space can be constrained or not - And finaly, the objective function may also change in time --- So there are many families of optimization problems, but there are also many families of optimization algorithms more or less adapted for these different problems. Gradient descent

Iteratively update $\boldsymbol{x}$ toward the steepest slope $$\boldsymbol{x} \leftarrow \boldsymbol{x} - \alpha \nabla_{\boldsymbol{x}}f(\boldsymbol{x})$$

Requires gradient of the objective function

  • $\alpha$ is the step size which influence the convergence rate
  • How to adapt it to converge faster ?

Popular variants of SGD in machine learning:

  • Momentum optimization: Nesterov Momentum / Nesterov Accelerated Gradient (NAG)
  • Adaptive methods: AdaGrad, RMSProp, Adam

Before talking about gradient free methods, let's quickly recall gradient descent. You already know this optimisation procedure [...], the gradient descent (or gradient ascent). You already used it to optimize weights in neural networks and you also used it in previous labs with Policy Gradient to optimize the policy of an agent. So very quickly, in gradient descent we iteratively update the current solution vector x toward the steepest slope. This requires to know the gradient on x of the objective function f. Here [...] alpha is the step size which influence the convergence rate And the classical question with these algorithms is “what is the best way to adapt the learning rate alpha in order to fasten the convergence” If the learning rate is to small, explored solutions from one iteration to the other will be very close and the convergence will be very slow ; and if it is too large, there may be no convergence at all. And to be efficient, the algorithm should adapt this rate along iterations, depending on the local shape of the objective function. So there are many variants of stochastic gradient descent, you probably saw some of them as they are very popular in Machine Learning There are two main families of variants of SGD to adapt the learning rate along iterations: - momentum methods with for instance the nesterov momentum algorithm - and the adaptive methods like adagrad or adam

Momentum methods

"Classic" Gradient Descent: $$\boldsymbol{x} \leftarrow \underbrace{\boldsymbol{x}}_{\text{position}} - \underbrace{\alpha ~ \nabla_{\boldsymbol{x}}f(\boldsymbol{x})}_{\text{momentum}}$$

Gradient Descent with momentum: $$\boldsymbol{m} \leftarrow \underbrace{\beta \boldsymbol{m}}_{\text{momentum}} - \underbrace{\alpha ~ \nabla_{\boldsymbol{x}}f(\boldsymbol{x})}_{\text{acceleration}}$$ $$\boldsymbol{x} \leftarrow \underbrace{\boldsymbol{x}}_{\text{position}} + \underbrace{\boldsymbol{m}}_{\text{momentum}}$$

This is just a recall, in momentum methods we introduce a momentum term to adapt the learning rate and fasten the convergence. The momentum term is a moving average of the past gradients - it is used to accelerate the convergence in directions where the gradient is consistent along iterations - and to slow down the convergence in directions where the gradient is not consistent along iterations.
