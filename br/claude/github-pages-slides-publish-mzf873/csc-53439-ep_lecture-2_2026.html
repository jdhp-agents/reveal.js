Exploration and Uncertainty

So we start with exploration.

Exploration-Exploitation Dilemma

  • Agent must explore to discover rewarding actions and states.
  • Agent must exploit known information to maximize reward.
  • Balancing exploration and exploitation is critical for efficient learning.
  • Too much exploration: slow progress; too much exploitation: suboptimal policies.
The exploration-exploitation dilemma is fundamental in RL: agents must balance trying new actions to gather information and leveraging what they know to maximize returns. This illustration is a good example: - we are used to a few restaurants (exploitation), we are happy with them, - but a new one just opened Do we try it (do we explore)? - with the risk of being disappointed, but maybe we will find a new favorite place? - or do we stick to what we know (exploitation)? --- "without enough explORAtion, we cannot learn the environment well enough; without enough explOItation, we cannot complete our reward optimization task" "Different RL algorithms balance between exploration and exploitation in different ways. In MC methods, Q-learning or many on-policy algorithms, the exploration is commonly implemented by ε-greedy;" "With exploitation, we take advantage of the best option we know. With exploration, we take some risk to collect information about unknown options. The best long-term strategy may involve short-term sacrifices. For example, one exploration trial could be a total failure, but it warns us of not taking that action too often in the future." --- - https://lilianweng.github.io/posts/2018-02-19-rl-overview/#exploration-exploitation-dilemma - https://lilianweng.github.io/posts/2018-01-23-multi-armed-bandit/#exploitation-vs-exploration

Unstructured Exploration

$\varepsilon$-greedy

Deep Q-Learning Algorithm
FOR EACH episode
$\quad$ $\mathbf{s} \leftarrow \text{env.reset}()$
$\quad$ DO
$\quad\quad$ $\mathbf{a} \leftarrow \color{red}{\epsilon\text{-greedy}(\mathbf{s}, \hat{Q}_{\mathbf{\omega_1}})}$
$\quad\quad$ $r, \mathbf{s'} \leftarrow \text{env.step}(\mathbf{a})$
$\quad\quad$ Store transition $(\mathbf{s}, \mathbf{a}, r, \mathbf{s'})$ in $\mathcal{D}$
$\quad\quad$ If $\mathcal{D}$ contains "enough" transitions
$\quad\quad\quad$ Sample random batch of transitions $(\mathbf{s}_j, \mathbf{a}_j, r_j, \mathbf{s'}_j)$ from $\mathcal{D}$ with $j=1$ to $m$
$\quad\quad\quad$ For each $j$, set $y_j = \begin{cases} r_j & \text{for terminal } \mathbf{s'}_j\\ r_j + \gamma \max_{\mathbf{a}^\star} \hat{Q}_{\mathbf{\omega_{2}}} (\mathbf{s'}_j)_{\mathbf{a}^\star} & \text{for non-terminal } \mathbf{s'}_j \end{cases}$
$\quad\quad\quad$ Perform a gradient descent step on $\left( y_j - \hat{Q}_{\mathbf{\omega_1}}(\mathbf{s}_j)_{\mathbf{a}_j} \right)^2$ with respect to the weights $\mathbf{\omega_1}$
$\quad\quad\quad$ Every $\tau$ steps reset $\hat{Q}_{\mathbf{\omega_2}}$ to $\hat{Q}_{\mathbf{\omega_1}}$, i.e., set $\mathbf{\omega_2} \leftarrow \mathbf{\omega_1}$
$\quad\quad$ $\mathbf{s} \leftarrow \mathbf{s'}$
$\quad$ UNTIL $\mathbf{s}$ is final
RETURN $\mathbf{\omega_1}$

At each step,:

  • with probability ε, select a random action;
  • with probability 1-ε, select the action with the highest estimated value.

👉 Simple to implement, widely used in discrete action spaces.

Typically, ε is annealed (decreased) over time to shift from exploration to exploitation.
The decay schedule is a hyperparameter and often determined empirically.

There are several kinds of exploration strategies. The most basic ones are unstructured exploration methods. Among them, ε-greedy is the most common in discrete action spaces. [...] Unstructured exploration uses simple stochasticity (randomness) to encourage the agent to try new actions. Effective in simple tasks, but often insufficient for complex environments. ε-greedy ensures some exploration but may revisit the same states/actions repeatedly and struggles in high-dimensional or sparse-reward settings.

Unstructured Exploration

Gaussian Noise

  • Add zero-mean Gaussian noise to the action output by the policy.
  • Common in algorithms for continuous control (e.g., DDPG, TD3).
  • Noise variance can be annealed to reduce exploration over time.
  • Encourages smooth, local exploration around the current policy.

$$\mu'(s_t) = \mu(s_t | \theta^{\mu}_t) + \mathcal{N}$$ where $\mu(s_t | \theta^{\mu}_t)$ is the actor network with weights $\theta^{\mu}_t$
and $\mathcal{N}$ is a noise process (e.g. Gaussian or Ornstein-Uhlenbeck).

In continuous action spaces, a common exploration strategy is to add Gaussian noise directly to the actions output by the policy network. This means that, instead of always taking the deterministic action suggested by the policy, we perturb it with random noise sampled from a normal distribution. This approach is widely used in algorithms like DDPG and TD3, where the action space is continuous. The variance of the noise can be annealed over time, so the agent explores more at the beginning of training and gradually exploits more as learning progresses. The main advantage is that it encourages smooth, local exploration around the current policy, which is often appropriate for physical control tasks. However, this method can be insufficient in environments where more global or structured exploration is needed, since the agent may only explore in a small neighborhood of its current behavior. In summary, Gaussian noise is simple and effective for many continuous control problems, but it may not be enough for tasks that require discovering distant or rare states.

Maximum Entropy RL

Exploration and Robustness via Entropy

Augments the reward with an entropy term to encourage diverse actions: $$ J(\pi) = \mathbb{E}_{\pi} \left[ \sum_t r_t + \alpha \mathcal{H}(\pi(\cdot|s_t)) \right] $$ where $\mathcal{H}$ is the entropy and $\alpha$ controls the exploration-exploitation tradeoff.

  • Promotes stochastic policies, improving robustness and exploration.
  • Widely used in robotics and continuous action domains.
Maximum-entropy RL augments the reward with an entropy term, encouraging stochastic policies and robust exploration. Soft Q-Learning and Soft Actor-Critic are leading algorithms in this area, especially effective for continuous control.

Intrinsic Reward-based Exploration

Augment environment reward $r_t$ with an intrinsic reward $r^i_t$: $$r_t \leftarrow r_t + \beta \cdot r^i_t$$ where $\beta$ balances exploitation vs exploration.

👉 Encourages agent to explore novel or uncertain states.

Two main approaches:

  • Count-based exploration: bonus for visiting rarely seen states.
  • Prediction-based exploration: bonus for high prediction error (surprise).
Intrinsic rewards is another approach to encourage exploration. Here, we augment the environment reward with an additional intrinsic reward signal to encourage exploration. This intrinsic reward can mean two things: novelty (count-based) or uncertainty/surprise (prediction-based). One common approach to better exploration, hard-exploration problem, is to augment the environment reward with an additional bonus signal to encourage extra exploration. The policy is thus trained with a reward composed of two terms, \(\; r_t = r_t^{e} + \beta\, r_t^{i} \;\), where \(\beta\) is a hyperparameter adjusting the balance between exploitation and exploration.
  • \( r_t^{e} \) is an extrinsic reward from the environment at time \( t \), defined according to the task at hand.
  • \( r_t^{i} \) is an intrinsic exploration bonus at time \( t \).
This intrinsic reward is somewhat inspired by intrinsic motivation in psychology (Oudeyer & Kaplan, 2008). Exploration driven by curiosity might be an important way for children to grow and learn. In other words, exploratory activities should be rewarding intrinsically in the human mind to encourage such behavior. The intrinsic rewards could be correlated with curiosity, surprise, familiarity of the state, and many other factors. Same ideas can be applied to RL algorithms. In the following sections, methods of bonus-based exploration rewards are roughly grouped into two categories:
  1. Discovery of novel states
  2. Improvement of the agent’s knowledge about the environment.

Intrinsic Reward-based Exploration

Count-based Exploration

Assigns exploration bonus inversely proportional to state visit count: $$r^i_t = \frac{1}{\sqrt{N(s_t)}}$$

Advantage: Effective in small, discrete state spaces.

Limitation: Infeasible in high-dimensional or continuous spaces
(counts become sparse or meaningless).

Alternatives: Density models approximate state visitation counts:

  • Bellemare et al., 2016 – CTS-based pseudocounts
  • Ostrovski et al., 2017 – PixelCNN-based pseudocounts
  • Tang et al., 2017 – Hash-based counts
If we consider intrinsic rewards as rewarding conditions that surprise us, we need a way to measure whether a state is novel or appears often. One intuitive way is to count how many times a state has been encountered and to assign a bonus accordingly. The bonus guides the agent’s behavior to prefer rarely visited states to common states. This is known as the count-based exploration method. Let \(N_{n}(s)\) be the empirical count function that tracks the real number of visits of a state \(s\) in the sequence of \(s_{1:n}\). Unfortunately, using \(N_{n}(s)\) for exploration directly is not practical, because most of the states would have \(N_{n}(s) = 0\), especially considering that the state space is often continuous or high-dimensional. Intrinsic Reward-based Exploration

Prediction-based Exploration

Uses prediction error as intrinsic reward: $$r^i_t = \| f(s_t, a_t) - s_{t+1} \|$$ where $f$ is a learned forward dynamics model.

Properties:

  • Agent is rewarded for visiting states where its model is uncertain or inaccurate (novelty/surprise).
  • Common in high-dimensional or continuous environments.

Alternatives: Density models approximate state visitation counts

  • Model can be "fooled" by stochasticity or noise.
  • May focus on unpredictable but irrelevant states.
  • Motivates use of random networks (e.g., RND) for more robust novelty estimation.
* The network tries to predict (s_{t+1}) (or a feature of (s_{t+1})) from (s_t, a_t). * The prediction difficulty is used as a novelty signal (error = curiosity). Learning a forward dynamics prediction model is a great way to approximate how much knowledge our model has obtained about the environment and the task MDPs. It captures an agent’s capability of predicting the consequence of its own behavior, \[ f : (s_t, a_t) \mapsto s_{t+1}. \] Such a model cannot be perfect (e.g., due to partial observation), the error \[ e(s_t, a_t) \;=\; \| f(s_t, a_t) - s_{t+1} \|^2_2 \] can be used for providing intrinsic exploration rewards. The higher the prediction error, the less familiar we are with that state. The faster the error rate drops, the more learning progress signals we acquire. https://lilianweng.github.io/posts/2020-06-07-exploration-drl/#forward-dynamics TODO: limites qui amènent aux random networks c.f. https://chatgpt.com/c/68ddabd1-09d8-8328-94a0-2e75d7fc3f0b

  • Agent is attracted to noisy or unpredictable regions, leading to biased and inefficient exploration.

Intrinsic Reward-based Exploration

Motivation for RND

A measure of novelty that is more stable and less sensitive to randomness.

  • Target network = a fixed random function \( f_{\text{target}}(s) \).
    • Since it is frozen, it defines a stable representation of states.
    • Each state has a unique but time-invariant signature.
  • Predictor network = a network trained to approximate \( f_{\text{target}}(s) \).
    • The prediction error is high at first for all states.
    • As some states are revisited, the predictor learns to approximate them → error decreases.
    • Truly novel states remain hard → high error → strong intrinsic reward.

👉 Unlike dynamics prediction, this error does not depend on the environment’s stochasticity, but only on whether the state has already been “learned” by the predictor.

RND provides a more stable and robust novelty measure by using a fixed target network and a predictor that learns to imitate it.

Intrinsic Reward-based Exploration

Focus on RND

  • RND uses two neural networks:
    • Target network: randomly initialized, fixed parameters.
    • Predictor network: trained to predict the target network's output for each state.
  • Intrinsic reward: prediction error (difference between predictor and target outputs).
    • States that are novel (rarely visited) yield high prediction error and thus high intrinsic reward.
    • As the agent visits a state more often, the predictor learns to match the target, reducing the intrinsic reward.
    • Encourages exploration of unfamiliar states.

Burda et al. (2018). Exploration by random network distillation.

### 📌 Advantages of this architecture 1. **Stability**: the target is frozen → no drift in the definition of novelty. 2. **No bias toward noise**: random states do not permanently yield high error, as the predictor eventually learns their fixed projection. 3. **Universality**: no knowledge of the environment’s dynamics is needed. 4. **Simplicity**: just two MLPs/CNNs, no need for complex transition models. --- ### 🎯 In summary * **We do not use next-state prediction** because it confuses novelty with unpredictability. * The **frozen target network** acts as a “pseudo-random representation generator” to give states a stable signature. * The **predictor network** measures how familiar a state is → and thus how much intrinsic reward it deserves.

Limits of Intrinsic Reward

Motivation for Memory-based Exploration

  • Intrinsic reward methods (e.g., novelty, prediction error) encourage exploration by rewarding new or surprising states.
  • Key limitations:
    • Slow function approximation: Neural networks may lag in updating, so states can appear "novel" longer than they are.
    • Non-stationary bonus: What is surprising changes as the agent learns, making the intrinsic reward unstable.
    • Knowledge fading: Once a state is no longer novel, the agent may forget its importance, even if it is crucial for solving the task.
  • Alternative: Memory-based exploration augments the agent with an explicit memory module to track visited states and contexts.
  • This enables more reliable novelty detection, stabilizes exploration signals, and helps retain knowledge of key states.
  • Example: Never Give Up (NGU) combines short-term episodic memory with long-term novelty signals (e.g., RND) to avoid forgetting useful states.
Les méthodes à récompense intrinsèque présentent trois limites majeures : lenteur de l’approximation par réseau de neurones (décalage entre la réalité et le signal), instabilité du bonus (non-stationnarité), et oubli de certains états importants (knowledge fading). Les approches à mémoire externe (memory-based exploration) permettent de stabiliser le signal d’exploration et de conserver la trace des états clés, comme dans Never Give Up (NGU).
Intrinsic reward approaches have key drawbacks: slow adaptation of neural approximators, non-stationary exploration bonuses, and forgetting of important states. Distributional RL

Categorical DQN (C51)

Introduction & Motivation

Classic DQN:

  • Learns only the expectation $Q(s,a)=\mathbb{E}[R \mid s,a]$
  • Bellman: \(Q(s,a) = R(s,a) + \gamma Q(s', a')\).
  • Lost information: variance, skewness, tail (risk) structure.

Distributional RL:

  • Model full return random variable \(Z(s,a)\) (not just its mean).
  • Distributional Bellman: \(\mathcal{T} Z(s,a) \stackrel{D}{=} R(s,a) + \gamma Z(s', a')\).
  • Benefits: richer training signal, more stable gradients, enables risk-sensitive objectives.

Key idea: preserve probabilistic structure throughout the backup,
then take \(\mathbb{E}[Z]\) only for action selection.

In this section, I’ll introduce the motivation behind distributional reinforcement learning, and specifically the C51 algorithm. Let’s start by recalling that classic algorithms like DQN aim to approximate the value of an action in a given state, that is, the expected future return. However, this approach collapses all uncertainty, variance, and risk structure into a single number. We lose all information about the spread of possible returns, the probability of very good or very bad outcomes, and the overall shape of the distribution—such as whether it’s skewed or has heavy tails. This is where distributional RL comes in. Instead of modeling only the expectation, we aim to approximate the full distribution of possible returns, denoted Z(s,a). This allows us to capture not just the mean, but also the variance, the tails, and thus to have a much richer picture of what the agent might experience. Mathematically, the Bellman operator becomes distributional: we no longer propagate a scalar value, but a probability law, which is shifted and contracted at each step. The C51 algorithm represents this distribution by discretizing it onto 51 fixed points, called “atoms,” and projects the updated distribution onto this support at every iteration. What are the benefits? First, the learning signal is much richer: instead of a single number, we supervise the agent with an entire distribution, which improves stability and convergence speed. Second, this opens the door to risk-sensitive policies: we can choose to optimize not just the mean, but also Value-at-Risk or other utility criteria. Finally, it’s important to note that even though action selection is still done via the expectation of the distribution, the entire learning phase benefits from this probabilistic structure, which explains the empirical gains observed. In summary, moving from expectation to the full distribution allows us to capture the richness of uncertainty, improve learning, and lay the groundwork for more robust and adaptive agents.

DQN Recap

  • Goal: approximate \( Q(s,a) = \mathbb{E}\left[\sum_{t} \gamma^t r_t \mid s,a\right] \)
  • Network: input is state, output is vector \( Q(s,a) \) for all actions
  • Bellman backup (target): \( y = r + \gamma \max_{a'} Q(s',a') \)
  • Loss: mean squared error \( (y - Q(s,a))^2 \)
  • Limitation: collapses all uncertainty into a single scalar value
Classic DQN uses scalar regression toward bootstrapped expectation; no distributional structure preserved.

Distributional RL

  • Random return: \( Z(s,a) \) is a random variable over \( \mathbb{R} \)
  • Distributional Bellman: \( \mathcal{T} Z(s,a) \stackrel{D}{=} r + \gamma Z(S', A') \), i.e., the law is transformed
  • Goal: approximate the law of \( Z(s,a) \) (not just \( \mathbb{E}[Z] \))
  • Expectation recoverable: \( Q(s,a) = \mathbb{E}[Z(s,a)] \)
  • C51: A Distributional Perspective on Reinforcement Learning, Bellemare et al., 2017.
Emphasize distributional Bellman operator and theoretical contraction in Wasserstein metric.

Categorical Approximation (C51)

  • Fixed discrete support: \( \{z_i\}_{i=0}^{N-1} \), with \( N = 51 \)
  • Bounded interval: \( [V_{\min}, V_{\max}] \)
  • \( \Delta z = \frac{V_{\max} - V_{\min}}{N-1} \)
  • Network predicts logits \( \to \) softmax \( \to \) \( p_i(s,a) \)
  • Approximate distribution: \( \mathbb{P}(Z(s,a) = z_i) = p_i \)
Choix fixe du support simplifie projection mais introduit rigidité si queue hors bornes.
Fixed categorical support enables simple cross-entropy optimization; clipping handles out-of-range targets.

C51 Architecture

  • Backbone identical to DQN: convolutional layers followed by fully connected layers.
  • Output: $(\#\text{actions} \times N)$ logits, reshaped to $(\#\text{actions}, N)$.
  • Softmax over atoms for each action: $p_i(s,a)$.
  • Action selection: $a^* = \arg\max_a \sum_i p_i(s,a) z_i$.
  • Target: uses frozen (target) network and projection onto fixed support.
Implementation detail: flatten logits, apply grouped softmax; expectation used for policy.

But wait...

Why model the full value distribution if action selection is based only on the expected value?

Key idea of the figure In Space Invaders, some actions ("Laser", etc.) often lead to immediate death → return ≈ 0. Others ("Left/Right/Noop") are "safe" → higher return (6–9). The return distribution is thus bimodal: a mass near 0 ("lose") + a higher mass ("survive"). If we only learn a scalar (Q(s,a)=\mathbb{E}[Z(s,a)]), we collapse these two events into a mean (e.g., 5.3). This mean can be "unrealizable" (we never actually observe 5.3: we either die or survive). As a result, each transition pulls the network in opposite directions (one update toward 0, one toward 8), leading to very "blurry" targets and contradictory gradients. In contrast, C51 learns the entire distribution (Z(s,a)) (a histogram over "atoms" z_i). The update distributes mass where it should go (closer to 0 after a death, higher after survival) without mixing these modes. This is what the text calls "keeping the 'lose' event separate from the 'survive' event". So, why model the distribution if the policy still uses argmax of the expectation? Even if we select actions using \mathbb{E}[Z] at evaluation, learning the distribution improves optimization: 1. More informative targets and more stable gradients. We align a probability vector (cross-entropy / categorical projection) rather than a single number (TD-error). Each transition updates all atoms → richer learning signal, lower variance, less oscillation due to mode mixing. 2. Better generalization under multimodality and heteroscedasticity. Two state/action pairs may have the same mean but very different risk. The distributional model does not merge these cases, so it generalizes better and propagates information more faithfully via bootstrapping. 3. Favorable theory. The distributional Bellman operator (T^\pi) is a contraction (in Wasserstein distance): we learn a mathematically well-posed object. (C51 adds a discrete projection that works very well in practice.) 4. "Free" bonus: risk & exploration. Even if C51 uses the expectation for acting, having Z(s,a) allows, if desired, risk-sensitive (quantiles, expectiles) or exploratory (optimism via variance) policies without retraining another network. In one sentence The policy may remain based on the expectation, but learning the distribution avoids the "blurriness" of the mean, provides better-conditioned gradients, and accelerates/stabilizes learning—which explains the empirical gains of C51. (texte original du papier Bellemare et al. 2017) This previously seen example illustrates the typical value distributions that can be observed. In this example, three actions (those including the button press) lead to the agent releasing its laser too early and eventually losing the game. The corresponding distributions reflect this: they assign a significant probability to 0 (the terminal value). The safe actions have similar distributions (LEFT, which tracks the invaders’ movement, is slightly favoured). This example helps explain why distributional RL approach is so successful: the distributional update keeps separated the low-value, “losing” event from the high-value, “survival” event, rather than average them into one (unrealizable) expectation.

DQN Algorithm

Input:
$\quad\quad$ none
Algorithm parameter:
$\quad\quad$ discount factor $\gamma$
$\quad\quad$ step size $\alpha \in (0,1]$
$\quad\quad$ small $\epsilon > 0$
$\quad\quad$ capacity of the experience replay memory $M$
$\quad\quad$ batch size $m$
$\quad\quad$ target network update frequency $\tau$

Initialize replay memory $\mathcal{D}$ to capacity $M$
Initialize action-value function $\hat{Q}_{\mathbf{\omega_1}}$ with random weights $\mathbf{\omega_1}$
Initialize target action-value function $\hat{Q}_{\mathbf{\omega_2}}$ with weights $\mathbf{\omega_2} = \mathbf{\omega_1}$

FOR EACH episode
$\quad$ $\mathbf{s} \leftarrow \text{env.reset}()$
$\quad$ DO
$\quad\quad$ $\mathbf{a} \leftarrow $$\epsilon\text{-greedy}(\mathbf{s}, \hat{Q}_{\mathbf{\omega_1}})$
$\quad\quad$ $r, \mathbf{s'} \leftarrow \text{env.step}(\mathbf{a})$
$\quad\quad$ Store transition $(\mathbf{s}, \mathbf{a}, r, \mathbf{s'})$ in $\mathcal{D}$
$\quad\quad$ If $\mathcal{D}$ contains "enough" transitions
$\quad\quad\quad$ Sample random batch of transitions $(\mathbf{s}_j, \mathbf{a}_j, r_j, \mathbf{s'}_j)$ from $\mathcal{D}$ with $j=1$ to $m$
$\quad\quad\quad$ For each $j$, set $y_j = \begin{cases} r_j & \text{for terminal } \mathbf{s'}_j\\ r_j + \gamma \max_{\mathbf{a}^\star} \hat{Q}_{\mathbf{\omega_{2}}} (\mathbf{s'}_j)_{\mathbf{a}^\star} & \text{for non-terminal } \mathbf{s'}_j \end{cases}$
$\quad\quad\quad$ Perform a gradient descent step on $\left( y_j - \hat{Q}_{\mathbf{\omega_1}}(\mathbf{s}_j)_{\mathbf{a}_j} \right)^2$ with respect to the weights $\mathbf{\omega_1}$
$\quad\quad\quad$ Every $\tau$ steps reset $\hat{Q}_{\mathbf{\omega_2}}$ to $\hat{Q}_{\mathbf{\omega_1}}$, i.e., set $\mathbf{\omega_2} \leftarrow \mathbf{\omega_1}$
$\quad\quad$ $\mathbf{s} \leftarrow \mathbf{s'}$
$\quad$ UNTIL $\mathbf{s}$ is final

RETURN $\mathbf{\omega_1}$

C51 Algorithm

FOR EACH episode
$\quad$ $\mathbf{s} \leftarrow \text{env.reset}()$
$\quad$ DO
$\quad\quad$ $\mathbf{a} \leftarrow $$\epsilon\text{-greedy}(\mathbf{s}, \hat{Z}_{\mathbf{\omega_1}})$
$\quad\quad$ $r, \mathbf{s'} \leftarrow \text{env.step}(\mathbf{a})$
$\quad\quad$ Store transition $(\mathbf{s}, \mathbf{a}, r, \mathbf{s'})$ in $\mathcal{D}$
$\quad\quad$ IF $\mathcal{D}$ contains "enough" transitions
$\quad\quad\quad$ Sample random batch of transitions $(\mathbf{s}_j, \mathbf{a}_j, r_j, \mathbf{s'}_j)$ from $\mathcal{D}$ with $j=1$ to $m$
$\quad\quad\quad$ FOR EACH transitions $(\mathbf{s}, \mathbf{a}, r, \mathbf{s'})$ of this batch
$\quad\quad\quad\quad$ Apply the Bellman operator
$\quad\quad\quad\quad$ Compute the projection $\Phi$ of $\mathcal{T}^{\pi}Z$ onto the support $\{z_i\}$
$\quad\quad\quad\quad$ Perform a gradient descent step to minimize the cross-entropy
$\quad\quad\quad$ Every $\tau$ steps reset $\hat{Z}_{\mathbf{\omega_2}}$ to $\hat{Z}_{\mathbf{\omega_1}}$, i.e., set $\mathbf{\omega_2} \leftarrow \mathbf{\omega_1}$
$\quad\quad$ $\mathbf{s} \leftarrow \mathbf{s'}$
$\quad$ UNTIL $\mathbf{s}$ is final

RETURN $\mathbf{\omega_1}$

$$Q(s,a) = \mathbb{E}[Z(s,a)] = \sum_i z_i , p_\omega(z_i \mid s,a).$$ The value $Q$ can be obtained as the expectation of $Z$.

Behavior policy: with probability $1-\epsilon$, take the action that maximizes the expected value
according to the current distributional estimate.

Since the support of the target distribution does not necessarily fall exactly on the fixed support of the predicted distribution,
we project it using the operator $(\Phi)$.

Perform a gradient descent step to minimize the cross-entropy between
the predicted distribution and the target distribution. $$L(\omega) = - \sum_i p_i^{\text{target}} \log p_i^{\text{pred}}$$

Hierarchical RL

Introduction

Human analogy: planning a trip vs. walking to the station

One of the things humans excel at is breaking down a complex task into smaller, more manageable subproblems. We solve these subtasks one by one and then combine their solutions to address the original challenge. Importantly, these subtasks often operate at different levels of abstraction compared to the overall goal. For example, imagine planning a trip from your home to a hotel room in a distant city. You don't think about every single foot step or body movement you need to take to reach the hotel. But that's what a classical reinforcement learning agent would do if it were tasked with this problem. Instead, at the highest level, you might decide which flight to book and which airport to land at. Each of these choices then leads to mid-level decisions, such as which train to take before or after the flight. Finally, these can be broken down into low-level actions, like choosing whether to take the metro, a bus, or a taxi to reach the hotel. This way of thinking—structuring decisions at multiple levels of abstraction—is precisely what Hierarchical Reinforcement Learning aims to capture. Hierarchical Reinforcement Learning, studies this real-world-inspired approach to problem solving.

The options framework

Sutton, R. S., Precup, D., & Singh, S. (1999). Between MDPs and semi-MDPs: A framework for temporal abstraction in reinforcement learning. Artificial Intelligence, 112(1-2), 181-211.

To formalize hierarchical reinforcement learning, we need a framework that can represent actions at multiple levels of abstraction. The options framework, introduced by Sutton et al. in 1999, provides such a formalism. It formalizes the concept of subgoals and subpolicies, allowing an agent to operate at different temporal scales.

The options framework

Motivation: Temporal Abstraction in RL

  • Standard MDPs only support primitive actions at each timestep.
  • Need: a way to represent and learn temporally extended behaviors.
  • Goal: extend RL framework minimally to include such temporal abstractions.

👉 Solution: Extend standard MDPs to support temporally extended actions called options

Standard MDPs only support primitive actions at each timestep. To do Hierarchical RL, we need a way to represent and learn temporally extended behaviors. The goal is extend RL framework to include such temporal abstractions but in a minimal way. A solution to this is the "options framework": it extends the standard MDP framework to include temporally extended actions called "options."

The options framework

What is an Option?

  • Some states are designated as subgoals
  • At a subgoal, the agent can choose an option (macro-action) instead of a primitive action
  • Options are sequences of primitive actions grouped together
  • Executing an option means executing this sequence of actions
The idea of options is simple : Some states in the environment are defined as "subgoals". When the agent reaches one of these subgoal states, it can choose to execute an "option" (or "macro-action") instead of a "primitive action". So an option is a sequence of primitive actions with grouped together. Executing an option means executing this sequence of actions.

The options framework

What is an Option?

An option $\omega$ is defined by a triple: $$\omega = \langle I, \pi, \beta \rangle$$

$I_{\omega}$ The initiation set $I \subseteq \mathcal{S}$ states where the option can be chosen.
$\pi_{\omega}(a|s)$ The subpolicy $\pi : \mathcal{S} \times \mathcal{A} \rightarrow [0, 1]$ controls which primitive actions are taken while the option runs (here a stochastic policy).
$\beta_{\omega}(s)$ The termination condition $\beta : \mathcal{S} \rightarrow [0, 1]$ tells us if $\omega$ terminates in $s$.
More formally, an option consists of three components: - An initiation set I: defines the states where the option can be initiated. - A subpolicy π: defines the behavior of the option. It is internal to the option. - A termination condition β: specifies when the option should terminate.

The options framework


The set of all options is denoted by $\Omega$.

The options framework

There are two kinds of policies in the options framework:

  • The meta-policy over options $\pi_{\Omega}(\omega|s)$, which selects which option to execute when the agent is in a state that belongs to the initiation set of one or more options.
  • The subpolicy of each option $\pi_{\omega}(a|s)$, which defines the behavior while the option is being executed.
There are two kinds of policies in the options framework: - The meta-policy over options, which selects which option to execute when the agent is in a state that belongs to the initiation set of one or more options. - The subpolicy of each option, which defines the behavior while the option is being executed.

Feudal Networks

Key Idea:

  • Inspired by Feudal RL (Dayan & Hinton, 1992): a Manager–Worker hierarchy.
  • Manager sets abstract goals in a latent space.
  • Worker executes primitive actions conditioned on these goals.

Tested on (Vezhnevets et al., 2016): Atari games

Achievements:

  • Learned effective temporal abstractions (macro-actions).
  • Showed faster learning and better exploration on hard sparse-reward environments compared to flat deep RL (DQN, A3C).
  • Demonstrated that latent goal vectors can guide behavior without explicit subgoal supervision.

Feudal Networks

Position Today:

  • Pioneered the goal-setting HRL paradigm.
  • However: rarely used in practice now
    • Instability in training.
    • Limited scalability to very complex tasks.
  • Key historical impact: introduced intrinsic goal representations and the idea of end-to-end differentiable hierarchy, still influential in modern HRL research.
