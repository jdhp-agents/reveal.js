Categorical DQN (C51)

Introduction & Motivation

Classic DQN:

  • Learns only the expectation $Q(s,a)=\mathbb{E}[R \mid s,a]$
  • Bellman: \(Q(s,a) = R(s,a) + \gamma Q(s', a')\).
  • Lost information: variance, skewness, tail (risk) structure.

Distributional RL:

  • Model full return random variable \(Z(s,a)\) (not just its mean).
  • Distributional Bellman: \(\mathcal{T} Z(s,a) \stackrel{D}{=} R(s,a) + \gamma Z(s', a')\).
  • Benefits: richer training signal, more stable gradients, enables risk-sensitive objectives.

Key idea: preserve probabilistic structure throughout the backup,
then take \(\mathbb{E}[Z]\) only for action selection.

In this section, I’ll introduce the motivation behind distributional reinforcement learning, and specifically the C51 algorithm. Let’s start by recalling that classic algorithms like DQN aim to approximate the value of an action in a given state, that is, the expected future return. However, this approach collapses all uncertainty, variance, and risk structure into a single number. We lose all information about the spread of possible returns, the probability of very good or very bad outcomes, and the overall shape of the distribution—such as whether it’s skewed or has heavy tails. This is where distributional RL comes in. Instead of modeling only the expectation, we aim to approximate the full distribution of possible returns, denoted Z(s,a). This allows us to capture not just the mean, but also the variance, the tails, and thus to have a much richer picture of what the agent might experience. Mathematically, the Bellman operator becomes distributional: we no longer propagate a scalar value, but a probability law, which is shifted and contracted at each step. The C51 algorithm represents this distribution by discretizing it onto 51 fixed points, called “atoms,” and projects the updated distribution onto this support at every iteration. What are the benefits? First, the learning signal is much richer: instead of a single number, we supervise the agent with an entire distribution, which improves stability and convergence speed. Second, this opens the door to risk-sensitive policies: we can choose to optimize not just the mean, but also Value-at-Risk or other utility criteria. Finally, it’s important to note that even though action selection is still done via the expectation of the distribution, the entire learning phase benefits from this probabilistic structure, which explains the empirical gains observed. In summary, moving from expectation to the full distribution allows us to capture the richness of uncertainty, improve learning, and lay the groundwork for more robust and adaptive agents.

DQN Recap

  • Goal: approximate \( Q(s,a) = \mathbb{E}\left[\sum_{t} \gamma^t r_t \mid s,a\right] \)
  • Network: input is state, output is vector \( Q(s,a) \) for all actions
  • Bellman backup (target): \( y = r + \gamma \max_{a'} Q(s',a') \)
  • Loss: mean squared error \( (y - Q(s,a))^2 \)
  • Limitation: collapses all uncertainty into a single scalar value
Classic DQN uses scalar regression toward bootstrapped expectation; no distributional structure preserved.

Distributional RL

  • Random return: \( Z(s,a) \) is a random variable over \( \mathbb{R} \)
  • Distributional Bellman: \( \mathcal{T} Z(s,a) \stackrel{D}{=} r + \gamma Z(S', A') \), i.e., the law is transformed
  • Goal: approximate the law of \( Z(s,a) \) (not just \( \mathbb{E}[Z] \))
  • Expectation recoverable: \( Q(s,a) = \mathbb{E}[Z(s,a)] \)
  • C51: A Distributional Perspective on Reinforcement Learning, Bellemare et al., 2017.
Emphasize distributional Bellman operator and theoretical contraction in Wasserstein metric.

Categorical Approximation (C51)

  • Fixed discrete support: \( \{z_i\}_{i=0}^{N-1} \), with \( N = 51 \)
  • Bounded interval: \( [V_{\min}, V_{\max}] \)
  • \( \Delta z = \frac{V_{\max} - V_{\min}}{N-1} \)
  • Network predicts logits \( \to \) softmax \( \to \) \( p_i(s,a) \)
  • Approximate distribution: \( \mathbb{P}(Z(s,a) = z_i) = p_i \)
Choix fixe du support simplifie projection mais introduit rigidité si queue hors bornes.
Fixed categorical support enables simple cross-entropy optimization; clipping handles out-of-range targets.

C51 Architecture

  • Backbone identical to DQN: convolutional layers followed by fully connected layers.
  • Output: $(\#\text{actions} \times N)$ logits, reshaped to $(\#\text{actions}, N)$.
  • Softmax over atoms for each action: $p_i(s,a)$.
  • Action selection: $a^* = \arg\max_a \sum_i p_i(s,a) z_i$.
  • Target: uses frozen (target) network and projection onto fixed support.
Implementation detail: flatten logits, apply grouped softmax; expectation used for policy.

But wait...

Why model the full value distribution if action selection is based only on the expected value?

Key idea of the figure In Space Invaders, some actions ("Laser", etc.) often lead to immediate death → return ≈ 0. Others ("Left/Right/Noop") are "safe" → higher return (6–9). The return distribution is thus bimodal: a mass near 0 ("lose") + a higher mass ("survive"). If we only learn a scalar (Q(s,a)=\mathbb{E}[Z(s,a)]), we collapse these two events into a mean (e.g., 5.3). This mean can be "unrealizable" (we never actually observe 5.3: we either die or survive). As a result, each transition pulls the network in opposite directions (one update toward 0, one toward 8), leading to very "blurry" targets and contradictory gradients. In contrast, C51 learns the entire distribution (Z(s,a)) (a histogram over "atoms" z_i). The update distributes mass where it should go (closer to 0 after a death, higher after survival) without mixing these modes. This is what the text calls "keeping the 'lose' event separate from the 'survive' event". So, why model the distribution if the policy still uses argmax of the expectation? Even if we select actions using \mathbb{E}[Z] at evaluation, learning the distribution improves optimization: 1. More informative targets and more stable gradients. We align a probability vector (cross-entropy / categorical projection) rather than a single number (TD-error). Each transition updates all atoms → richer learning signal, lower variance, less oscillation due to mode mixing. 2. Better generalization under multimodality and heteroscedasticity. Two state/action pairs may have the same mean but very different risk. The distributional model does not merge these cases, so it generalizes better and propagates information more faithfully via bootstrapping. 3. Favorable theory. The distributional Bellman operator (T^\pi) is a contraction (in Wasserstein distance): we learn a mathematically well-posed object. (C51 adds a discrete projection that works very well in practice.) 4. "Free" bonus: risk & exploration. Even if C51 uses the expectation for acting, having Z(s,a) allows, if desired, risk-sensitive (quantiles, expectiles) or exploratory (optimism via variance) policies without retraining another network. In one sentence The policy may remain based on the expectation, but learning the distribution avoids the "blurriness" of the mean, provides better-conditioned gradients, and accelerates/stabilizes learning—which explains the empirical gains of C51. DQN Algorithm

Input:
$\quad\quad$ none
Algorithm parameter:
$\quad\quad$ discount factor $\gamma$
$\quad\quad$ step size $\alpha \in (0,1]$
$\quad\quad$ small $\epsilon > 0$
$\quad\quad$ capacity of the experience replay memory $M$
$\quad\quad$ batch size $m$
$\quad\quad$ target network update frequency $\tau$

Initialize replay memory $\mathcal{D}$ to capacity $M$
Initialize action-value function $\hat{Q}_{\mathbf{\omega_1}}$ with random weights $\mathbf{\omega_1}$
Initialize target action-value function $\hat{Q}_{\mathbf{\omega_2}}$ with weights $\mathbf{\omega_2} = \mathbf{\omega_1}$

FOR EACH episode
$\quad$ $\mathbf{s} \leftarrow \text{env.reset}()$
$\quad$ DO
$\quad\quad$ $\mathbf{a} \leftarrow $$\epsilon\text{-greedy}(\mathbf{s}, \hat{Q}_{\mathbf{\omega_1}})$
$\quad\quad$ $r, \mathbf{s'} \leftarrow \text{env.step}(\mathbf{a})$
$\quad\quad$ Store transition $(\mathbf{s}, \mathbf{a}, r, \mathbf{s'})$ in $\mathcal{D}$
$\quad\quad$ If $\mathcal{D}$ contains "enough" transitions
$\quad\quad\quad$ Sample random batch of transitions $(\mathbf{s}_j, \mathbf{a}_j, r_j, \mathbf{s'}_j)$ from $\mathcal{D}$ with $j=1$ to $m$
$\quad\quad\quad$ For each $j$, set $y_j = \begin{cases} r_j & \text{for terminal } \mathbf{s'}_j\\ r_j + \gamma \max_{\mathbf{a}^\star} \hat{Q}_{\mathbf{\omega_{2}}} (\mathbf{s'}_j)_{\mathbf{a}^\star} & \text{for non-terminal } \mathbf{s'}_j \end{cases}$
$\quad\quad\quad$ Perform a gradient descent step on $\left( y_j - \hat{Q}_{\mathbf{\omega_1}}(\mathbf{s}_j)_{\mathbf{a}_j} \right)^2$ with respect to the weights $\mathbf{\omega_1}$
$\quad\quad\quad$ Every $\tau$ steps reset $\hat{Q}_{\mathbf{\omega_2}}$ to $\hat{Q}_{\mathbf{\omega_1}}$, i.e., set $\mathbf{\omega_2} \leftarrow \mathbf{\omega_1}$
$\quad\quad$ $\mathbf{s} \leftarrow \mathbf{s'}$
$\quad$ UNTIL $\mathbf{s}$ is final

RETURN $\mathbf{\omega_1}$

C51 Algorithm

FOR EACH episode
$\quad$ $\mathbf{s} \leftarrow \text{env.reset}()$
$\quad$ DO
$\quad\quad$ $\mathbf{a} \leftarrow $$\epsilon\text{-greedy}(\mathbf{s}, \hat{Z}_{\mathbf{\omega_1}})$
$\quad\quad$ $r, \mathbf{s'} \leftarrow \text{env.step}(\mathbf{a})$
$\quad\quad$ Store transition $(\mathbf{s}, \mathbf{a}, r, \mathbf{s'})$ in $\mathcal{D}$
$\quad\quad$ IF $\mathcal{D}$ contains "enough" transitions
$\quad\quad\quad$ Sample random batch of transitions $(\mathbf{s}_j, \mathbf{a}_j, r_j, \mathbf{s'}_j)$ from $\mathcal{D}$ with $j=1$ to $m$
$\quad\quad\quad$ FOR EACH transitions $(\mathbf{s}, \mathbf{a}, r, \mathbf{s'})$ of this batch
$\quad\quad\quad\quad$ Apply the Bellman operator
$\quad\quad\quad\quad$ Compute the projection $\Phi$ of $\mathcal{T}^{\pi}Z$ onto the support $\{z_i\}$
$\quad\quad\quad\quad$ Perform a gradient descent step to minimize the cross-entropy
$\quad\quad\quad$ Every $\tau$ steps reset $\hat{Z}_{\mathbf{\omega_2}}$ to $\hat{Z}_{\mathbf{\omega_1}}$, i.e., set $\mathbf{\omega_2} \leftarrow \mathbf{\omega_1}$
$\quad\quad$ $\mathbf{s} \leftarrow \mathbf{s'}$
$\quad$ UNTIL $\mathbf{s}$ is final

RETURN $\mathbf{\omega_1}$

$$Q(s,a) = \mathbb{E}[Z(s,a)] = \sum_i z_i , p_\omega(z_i \mid s,a).$$ The value $Q$ can be obtained as the expectation of $Z$.

Behavior policy: with probability $1-\epsilon$, take the action that maximizes the expected value
according to the current distributional estimate.

Since the support of the target distribution does not necessarily fall exactly on the fixed support of the predicted distribution,
we project it using the operator $(\Phi)$.

Perform a gradient descent step to minimize the cross-entropy between
the predicted distribution and the target distribution. $$L(\omega) = - \sum_i p_i^{\text{target}} \log p_i^{\text{pred}}$$

Limitations & Extensions

  • Fixed support: risk of saturation if returns fall outside $[V_{\min}, V_{\max}]$
  • Uniform resolution: inefficient for modeling sharp tails or skewed distributions
  • Extensions:
    • QR-DQN: learns quantiles, reducing projection bias
    • IQN: continuous implicit quantile function for flexible modeling
  • Paradigm: working with distributions enables risk-sensitive and utility-based RL
Limitations of fixed categorical grid motivate quantile-based and adaptive approaches.

C51

Conclusion & Discussion

  • Moving from expectation to distribution provides a richer learning signal.
  • C51 is simple, effective, and forms the basis for many improvements.
  • Actions are selected via the expectation: the full distribution is valuable for training and for future risk-sensitive objectives.
  • Enables risk-sensitive RL, robustness, and informed exploration.
  • Open questions: how to choose the support? How to combine with epistemic uncertainty?
Even if acting greedily on expectation, learning benefits from structured multi-target supervision; distribution reusable for risk measures.

Quantile Regression Deep Q-Network

  • Represents the value distribution by estimating a fixed set of quantiles.
  • Uses quantile regression loss to train the network.
  • Improves stability and expressiveness over categorical methods.

Distributional Reinforcement Learning with Quantile Regression, Dabney et al., 2017.

QR-DQN models the return distribution using quantiles, enabling more precise and robust value estimation than fixed categorical bins.
